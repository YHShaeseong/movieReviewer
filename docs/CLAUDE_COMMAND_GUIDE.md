# Claude 명령 가이드

단계별 학습하면서 개발하기 위한 효과적인 명령 방법

## 핵심 원칙

**"작은 단위로 + 설명과 함께 + 하나씩"**

---

## 1. 범위를 명확히 지정

```
❌ "팝업 만들어줘" (너무 포괄적)

✅ "팝업의 HTML 구조만 먼저 만들어줘"
✅ "팝업 스타일링만 CSS로 작성해줘"
✅ "팝업 열고 닫는 JavaScript 기능만 추가해줘"
```

## 2. 설명 요청하기

```
✅ "각 코드가 무슨 역할인지 주석으로 설명해줘"
✅ "이 코드를 추가하는 이유를 설명해줘"
✅ "CSS 속성이 어떻게 동작하는지 설명해줘"
```

## 3. 단계별로 진행

```
✅ "1단계: 팝업 HTML 뼈대만 만들어줘"
✅ "2단계: 팝업이 화면 중앙에 오도록 CSS만 추가해줘"
✅ "3단계: 닫기 버튼 기능만 JavaScript로 구현해줘"
```

## 4. 기능 단위로 분리

```
✅ "질문 1개만 추가해줘 (선호 장르 선택)"
✅ "선택한 장르를 저장하는 기능만 추가해줘"
✅ "저장된 데이터를 콘솔에 출력하는 기능만 추가해줘"
```

## 5. "~만" 키워드 활용

```
✅ "CSS만", "HTML만", "JavaScript만"
✅ "스타일만", "구조만", "기능만"
✅ "이 부분만", "여기만 수정해줘"
```

## 6. 예제 먼저 요청

```
✅ "간단한 예제 코드로 먼저 보여줘"
✅ "이 개념을 설명할 수 있는 최소 코드만 작성해줘"
```

---

## 추천 명령 템플릿

```
"[기능명]의 [HTML/CSS/JS] [구조/스타일/기능]만 작성해줘.
각 부분이 어떤 역할인지 주석으로 설명해줘."
```

### 실전 예시

```
✅ "팝업의 HTML 구조만 작성해줘. 각 div가 어떤 역할인지 주석으로 설명해줘."

✅ "팝업이 화면 중앙에 표시되는 CSS만 추가해줘. position과 transform이 어떻게 동작하는지 설명해줘."

✅ "닫기 버튼 클릭 시 팝업이 사라지는 JavaScript 기능만 추가해줘."
```

---

## 작업 흐름 예시

### 팝업 기능 개발 과정

1. "팝업 HTML 구조만 만들어줘"
2. "팝업 기본 스타일(크기, 위치)만 CSS로 추가해줘"
3. "팝업 오버레이 어둡게 하는 스타일만 추가해줘"
4. "페이지 로드 시 팝업 표시하는 JS 기능만 추가해줘"
5. "닫기 버튼 클릭 시 팝업 닫히는 기능만 추가해줘"
6. "첫 번째 질문(장르 선택) HTML만 추가해줘"
7. "선택한 장르 저장하는 기능만 추가해줘"

---

## 주의사항

- 한 번에 여러 기능 요청하지 않기
- "전체적으로", "다 만들어줘" 같은 표현 피하기
- 이해 안 되면 바로 "이 부분 설명해줘" 요청하기
- 다음 단계 진행 전에 현재 코드 이해하고 넘어가기
